# つぶナビ デモ用サンプルデータ

## 使い方
1. `home_tsubunavi.html` を開く
2. 以下のつぶやきを順番にコピー＆ペーストして「つぶやく」ボタンをクリック
3. タスクボードに自動登録されるのを確認
4. タスクをドラッグ＆ドロップで「完了」に移動
5. 「日報生成」ボタンをクリックして自動生成された日報を確認

---

## サンプルつぶやき（順番にコピペしてください）

### 1. 朝のつぶやき（ポジティブ＋タスク）
```
おはようございます！今日は新しいAPI設計書を作成する予定です。楽しみ！
```
→ **期待される動作**: 
- 「API設計書を作成」がタスクボードの「待機中」に追加
- 📋業務タスク、😊ポジティブのタグが付く

## タスク完了のデモ手順

### ステップ1: タスクを「実施中」に移動
1. 「API設計書を作成」のカードをドラッグ
2. 「🔄 実施中」のカラムにドロップ

### ステップ2: タスクを「完了」に移動
1. 「API設計書を作成」のカードをドラッグ
2. 「✅ 完了」のカラムにドロップ
3. **レベルアップアニメーション**が表示される
4. スキルレベルが +5pt 上昇
5. 風船🎈が飛ぶアニメーション

### ステップ3: 他のタスクも完了させる
- 「チーム会議の準備」→ 完了
- 「データベース設計の調査」→ 完了

## デモのポイント

### 1. AIタスク整形
- つぶやきからタスクを自動抽出
- **AIが分かりやすいタスク名に変換**
  - 例: 「新しいAPI設計書を作成する予定」→ 「**API設計書の作成**」
  - 例: 「チーム会議の準備をしないと」→ 「**チーム会議の準備**」
  - 例: 「調査を進めて解決策を見つけないと」→ 「**解決策の調査**」

### 2. 感情分析
- ポジティブ: 「嬉しい」「楽しい」「良い」「成功」「できた」
- ネガティブ: 「難しい」「困」「大変」「疲」「できない」

### 3. 日報の自動要約
- タスクをカテゴリ別に分類（設計、実装、会議、調査など）
- ポジティブ/ネガティブな感情を考慮した気づきを生成

### 4. スキル成長の可視化
- タスク完了時にレベルアップ
- レーダーチャートが更新
- 風船アニメーションで達成感を演出

## トラブルシューティング

### タスクが追加されない場合
- 「〜する」「〜作成」「〜対応」などの動詞を含めてください
- 例: ❌「API設計書」→ ⭕「API設計書を作成する」

### 日報が生成されない場合
- 最低1つはつぶやきを投稿してください
- ブラウザのコンソールでエラーを確認してください

### タスクがドラッグできない場合
- カードの中央部分をクリック＆ドラッグしてください
- ブラウザを更新してみてください
